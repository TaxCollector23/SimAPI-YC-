The `simapi` CLI ships with the Node package and validates simulation files from
the terminal or a CI pipeline. Requires Node 18+.

## Install

```bash
npm install -g simapi
```

## Authenticate

```bash
simapi login        # prompts for your key, saves to ~/.simapi/config.json
```

`SIMAPI_API_KEY` overrides the saved key — convenient in CI.

## Commands

<CodeGroup>

```bash Validate
simapi init                       # write a sample simulation.json
simapi validate simulation.json
simapi validate run.json --type structural
```

```bash JSON output
simapi validate run.json --json | jq .status
```

```bash CI gate (exit code)
simapi validate run.json --fail-on warning
# exit 0 = passed · exit 1 = warning/failed
```

```bash Session
simapi whoami
simapi logout
simapi version
simapi help
```

</CodeGroup>

## Output

```text
  Status:     PASSED
  Confidence: high
  Trials:     196/200 valid
  Checks:     287 (0 failed, 4 warnings)
  Time:       23.4ms
```

## Options

<ParamField path="--type <domain>" type="string" default="aerodynamics">Simulation domain.</ParamField>
<ParamField path="--json" type="flag">Print the raw JSON response.</ParamField>
<ParamField path="--no-ai" type="flag">Skip the async AI layer.</ParamField>
<ParamField path="--fail-on <level>" type="string">Exit non-zero on `warning` or `failed`.</ParamField>

## Environment

<ParamField path="SIMAPI_API_KEY" type="string">Overrides the saved key.</ParamField>
<ParamField path="SIMAPI_BASE_URL" type="string" default="https://sim-api.vercel.app/api">API base URL.</ParamField>

<Tip>
  Use `--fail-on` to turn a validation verdict into a CI exit code — the
  foundation for [gating your pipeline](/cicd/github-actions).
</Tip>
